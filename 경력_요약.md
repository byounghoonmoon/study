- 본인의 단점  
    1. 집중이 되었을 경우 헤어 나오지를 못한다      
<br>
- 프로젝트를 하던 중 가장 힘들었던 점과 그것을 어떻게 극복하였는지 ? 
    1. 메트라이프 글로벌에서 만든 Framework을 사용하는 프로젝트인데 Framework 문서가 존재하지 않아 이해하기가 어려웠습니다. 수정 요청을 위해선 수많은 프로세스를 거쳐야 하며 시간이 오래 걸려 다른 방법으로 대체 하기도 하였습니다.    

    2. 인력 부족으로 인한 업무과중이 힘들었습니다. 특히, 단기 프로젝트의 경우 TA 인력을 구하는것은 더더욱 힘들었습니다. 프로젝트 오픈일은 지났지만 인력이 구해지지 않아, TA 업무를 병행하였습니다. 처음 접한 내용도 있었지만 찾아보며 해 나가며 재미도 느꼈습니다. 다만 이렇게 다중적인 일을 하다보니 시간은 부족하고 자연스럽게 야근을 하게 되어 몸이 힘들었습니다. 덕분에 배운 내용도 많지만 몸이 많이 힘들었던 기억이 납니다.  
<br>
- Spring Cloud에서 Zuul 에대해 설명하시오  
    1. Zuul 은 BackEnd에 도달 하기전 거치는 Gateway 역할을 담당하며, 주된 역할로는 동적 라우팅 , 모니터링 , 보안 등이 있습니다. 
    2. Zuul의 중심에는 다양한 필터가 존재하며 이런 필터들을 통해 http 요청에서 응답까지의 라우팅 및 다양한 작업이 가능 합니다. 기본 Type 필터에는 "PRE, ROUTE, POST, ERROR"가 있습니다. 
    3. Zuul의 경우 모든 Request가 집중되므로, 장애가 일어나면 모든 서비스가 동작하지 않을 수 있습니다. 따라서, Retry와 LoadBalancing을 할 수 있는 Ribbon과 장애 전파를 막아주는 Hystrix가 내부 디펜던시로 되어있습니다. 
    4. Ribbon은 Client-Side 로드밸런싱 역할을 담당. 서버 대상 목록은 Eureka를 기반으로 동적으로 설정 가능하며 서버로 부터 정상적인 응답을 받지 못하였을 경우 Retry 하는 기능이 주된 기능으로 볼 수 있습니다. 
    5. Hystrix는 연쇄 장애를 방지하는 역할으로 볼 수 있습니다. Thread-Pool, Queue, Semaphore , circuit Open 상태를 확인하여 fallback을 호출 합니다.  
<br>
- Spring Cloud ConfigServer에 대하여 설명하시오
    1. 각 애플리케이션에의 Config 설정을 중앙 서버에서 관리를 하는 서비스입니다.
    2. 서버 재기동을 하지 않고 "/actuator/refresh", "/actuator/busrefresh" 동작을 통해 설정 값을 변경할 수 있습니다.  
<br>
- CI/CD에 대해 설명하고 구축경험을 말해보시오
    1. CI/CD(Continuous Integration & Continuous Deployement)란 지속적 통합 및 배포의 준말입니다. CI는 빌드 및 테스트 과정을 자동화 해주는 역할이며, CD는 배포를 자동화 해주는것이라고 쉽게 생각 할 수 있습니다. 

    2. 아틀라시안 그룹의 Bamboo 제품을 통하여 CI/CD를 구축한 경험이 있습니다. 레파지토리로는 BitBucket을 사용하였고, 코드 인스펙션의 경우 SonarQube 를 사용하였습니다. 빌드 서버에서는 NPM을 처음 도입하였기 때문에 Proxy 설정 등을  해주었습니다.  
    프로세스는 Build → Test → Code Inspection → Deploy 로 진행하였습니다.
    Bamboo Server에서 배포 대상 서버 접속을 위한 SSH 작업,
    서버 중지 및 기동 스크립트 작성과 같은 업무를 수행 하였습니다. 
    기억에 남는 경험으로는 3개 서버가 재기동 정상 유무와 상관없이 재기동 되는 버그가 있었는데, 순차 재기동 할 수 있도록 Python 스크립트를 작성한 경험이 있습니다. /actuator/health 호출을 통하여 status가 정상일 경우에만 다음 서버를 재기동하도록 만들었습니다. 파이썬 스크립트 상수로 "헬스체크 URL 호출 반복 횟수 및 간격시간"을 두었습니다.
